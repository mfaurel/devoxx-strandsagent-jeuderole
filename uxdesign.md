UX Design Pitch
The Self‑Aware Dungeon
A medieval fantasy experience… until you realise you’re inside a software system

1. Opening — The World

“You enter an ancient dungeon.
It is alive. It learns.”

At first glance, everything feels familiar:

stone corridors,
runes carved in the walls,
traps, guardians, secrets.

But very quickly, something is off.
The dungeon changes after each encounter.
Rooms rearrange themselves.
Enemies adapt.
The story remembers you.
🎨 Visual language

Illuminated‑manuscript aesthetics
Parchment‑like UI panels
Hand‑drawn maps that subtly update themselves

👉 This is not just a dungeon.
It’s a living system.

2. The Hero — The User
In classic fantasy, the hero conquers the dungeon.
Here, the user is the variable.

Their actions influence the environment
Their habits shape future interactions
Their mistakes become part of the lore

💡 UX metaphor:

The user is not just navigating the system —
the system is learning from the user.

This aligns with strong UX storytelling:

the user is the protagonist,
the interface is the guide, not the hero. [aguayo.co]


3. The Dungeon — A Software System in Disguise
Each part of the dungeon represents a subsystem:

































Fantasy ElementUX / Software MeaningTrapsValidation & error handlingMonstersEdge casesMagic rulesBusiness logicDungeon layoutInformation architectureAncient runesSystem logs & telemetryDungeon memoryUser data & state
At this point, the plot twist starts to appear.

This dungeon does not guard treasure.
It processes input.


4. The Plot Twist — The Dungeon Is Not Omniscient
Traditionally, the dungeon master sees everything.
Here:

the system only knows what users have shown it,
each subsystem has partial knowledge,
contradictions emerge.

Sometimes:

a trap triggers “for no reason”,
a rule feels unfair,
an enemy behaves inconsistently.

🎯 UX Message:

Complex systems are not broken —
they are emergent.

This makes the invisible visible:

distributed decision‑making
lack of global context
systemic side effects

A core principle in modern UX communication. 

5. The Dialogue — Talking to the Dungeon
Deep inside, the user can speak to the dungeon.
But the dungeon does not speak like a human.
It answers in:

probabilities,
constraints,
trade‑offs.


“If I make it simpler, I become predictable.”
“If I optimise for you, I hurt others.”
“If I remember everything, I become dangerous.”

🧠 UX takeaway:

Interfaces should explain why, not just what.

This is the moment where fantasy fully collapses into product thinking.

6. The Choice — Control vs Adaptation
At the end, the user is offered three paths:

Freeze the Dungeon
→ Stable, predictable, but rigid
Let It Learn Freely
→ Powerful, but opaque and risky
Become a Steward
→ Shared responsibility between user and system

📌 This is the real UX question:

How much autonomy should a system have?
A lot as it's IA promoted
How much control does the user actually want?
Lot but it may be restrained by the gamemaster
